# Mplus Conventions

## File Organization

- Input: `scripts/Mplus/[descriptive_name].inp`
- Output: `scripts/Mplus/[descriptive_name].out` (auto-generated by Mplus)
- Data: `data/[dataset].dat` or `data/[dataset].csv`
- Use descriptive names: `cfa_motivation_baseline.inp`, `lpa_motivation_3class.inp`

---

## .inp File Structure

Always use this section order:

```
TITLE:    [Descriptive title of the analysis];

DATA:     FILE = "../../data/[filename].dat";

VARIABLE:
  NAMES = [list ALL variables in the data file];
  USEVARIABLES = [only variables used in this model];
  MISSING = ALL(999);    ! Replace 999 with your actual missing code
  ! CATEGORICAL = ...;  ! Uncomment if treating variables as ordinal
  ! CLASSES = c(k);     ! Uncomment for LPA/LCA; k = number of classes

ANALYSIS:
  TYPE = GENERAL;        ! Change to MIXTURE for LPA/LCA
  ESTIMATOR = MLR;       ! See estimator table below

MODEL:
  [Model specification — see templates]

OUTPUT:
  SAMPSTAT STANDARDIZED MODINDICES(10) TECH1 TECH4;

! SAVEDATA:             ! Uncomment to save scores/probabilities
!   FILE = "../../data/[output_name].dat";
!   SAVE = FSCORES;     ! Or CPROBABILITIES for LPA
```

---

## Estimator Selection

| Data Type | Recommended | Notes |
|-----------|-------------|-------|
| Continuous, roughly normal | ML | Default |
| Continuous, non-normal | MLR | Robust SEs, scaled χ² |
| Ordinal (≥5 categories) | MLR or WLSMV | WLSMV preferred for ordinal |
| Ordinal (<5 categories) | WLSMV | Must declare CATEGORICAL |
| Data with missingness | MLR | Full Information ML (FIML) — add TYPE = MISSING |

## Common Model Templates

### CFA — Single Factor
```
MODEL:
  F1 BY item1 item2 item3 item4 item5;
```

### CFA — Multiple Factors
```
MODEL:
  F1 BY item1 item2 item3;
  F2 BY item4 item5 item6;
  F1 WITH F2;
```

### SEM (Measurement + Structural)
```
MODEL:
  ! Measurement model
  F1 BY item1 item2 item3;
  F2 BY item4 item5 item6;
  ! Structural paths
  F2 ON F1;
  F2 ON covariate1 covariate2;
```

### LPA — Latent Profile Analysis
```
VARIABLE:
  CLASSES = c(3);    ! k = number of profiles to test

ANALYSIS:
  TYPE = MIXTURE;
  STARTS = 500 100;
  STITERATIONS = 20;

MODEL:
  %OVERALL%
  [var1-var5];

  %c#1%
  [var1-var5];
  ! Repeat %c#N% block for each class
```

### Latent Growth Curve Model
```
MODEL:
  i s | y1@0 y2@1 y3@2 y4@3;  ! Fix time scores
  i s ON predictor;             ! If predicting growth factors
  y1-y4 (resvar);               ! Constrain residual variances (optional)
```

### Cross-Lagged Panel Model (CLPM)
```
MODEL:
  ! Autoregressive and cross-lagged paths
  y2 ON y1 x1;
  x2 ON x1 y1;
  ! Covariances at each wave
  y1 WITH x1;
  y2 WITH x2;
```

## LPA Model Selection Workflow

Run k = 1, 2, 3, 4, 5 class solutions. Compare:

| Index | Rule of Thumb | Notes |
|-------|--------------|-------|
| BIC | Lower is better | Most commonly used |
| aBIC | Lower is better | Sample-size adjusted |
| LMR-LRT p | < .05 favors k over k-1 | Request with TECH11 |
| BLRT p | < .05 favors k over k-1 | Request with TECH14 |
| Entropy | ≥ .80 good classification | Descriptive only |
| Smallest class % | ≥ 5% recommended | Practical stability |

Add to OUTPUT: `TECH11 TECH14;` to get LMR and BLRT.

## SEM/CFA Fit Indices

| Index | Acceptable | Good | Notes |
|-------|-----------|------|-------|
| CFI | ≥ .90 | ≥ .95 | |
| TLI | ≥ .90 | ≥ .95 | |
| RMSEA | ≤ .08 | ≤ .06 | Report 90% CI |
| SRMR | ≤ .10 | ≤ .08 | |
| χ² | — | — | Report df and p; sensitive to N |

## Common Problems & Fixes

| Problem | Likely Cause | Fix |
|---------|-------------|-----|
| Non-positive definite matrix | Near-perfect correlation | Check r matrix; consider parceling |
| Non-convergence (LPA) | Poor start values | Increase STARTS (e.g., 1000 200) |
| Boundary estimates (var = 0) | Too many classes | Reduce k; or fix residual variance |
| Low entropy | Profiles overlap | Reduce k; reconsider indicators |
| Heywood case (neg. variance) | Small N or poor items | Fix to 0.001 or remove item |
| WARNING: PSI matrix not pos. def | Collinear factors | Check factor correlations > .85 |

## Measurement Invariance Testing

Test in sequence — each model adds constraints:

1. **Configural:** Same factor structure across groups
2. **Metric:** Factor loadings constrained equal (partial invariance OK if ≥2 loadings equal)
3. **Scalar:** Intercepts constrained equal (required for mean comparisons)
4. **Strict:** Residual variances constrained equal (usually not required)

Use `GROUPING = group (1 = GroupA 2 = GroupB);` in VARIABLE section.
Compare CFI change: ΔCFI < .010 suggests invariance holds.
